具体物料打分走查（演示）

标准库（内存索引）含 3 个分类，tokenIndex 按名称建立：

  • C1 编码 B07 名称「不锈钢板」（同义词含「304」）
  • C2 编码 B08 名称「不锈钢」
  • C3 编码 B09 名称「钢板」

待检测物料（为演示分词效果，物料名称中「不锈钢 / 钢板」作为独立词出现）：

  • 系统分类编码 = B07，系统分类名称 = 不锈钢板
  • 物料名称 = 不锈钢 钢板，规格 = 304，牌号 = 空
步骤调用文本/编码命中分类权重该分类 bump 后(取 max)
1scoreCodecategoryCode=B07C11.0C1 = 1.0
2scoreCodematerialCode(无)—0.8无命中
3scoreTextcategoryName=不锈钢板C1(精确+分词)0.9 / 0.54C1 = max(1.0,0.9,0.54) = 1.0
4scoreTextmaterialName=不锈钢 钢板C2(不锈钢), C3(钢板)0.85×0.6=0.51C2 = 0.51，C3 = 0.51
5scoreTextspecification=304C1(同义词304)0.6×0.6=0.36C1 = max(1.0,0.36) = 1.0
6scoreTextgrade(空)—0.5无命中

排序结果：C1(1.0) > C2(0.51) = C3(0.51)。topK=10 时三者全部作为候选返回，喂给大模型做最终判断。

中文分词局限（重要）：当前 tokenize 只按「非(字母/数字/中文)」切分，纯中文连续串不会被子串拆分。 例如物料名「碳素结构钢」只会产生一个 token「碳素结构钢」，无法召回名称「结构钢」或「碳素钢」的分类。 因此召回对中文长名称的泛化能力有限——物料名最好含可切分的英文/数字/分隔符，或标准库补充同义词来弥补。
